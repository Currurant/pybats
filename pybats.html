pybats.analysis module¶

pybats.analysis.analysis(Y, X, k, forecast_start, forecast_end, nsamps=500, family='normal', n=None, model_prior=None, prior_length=20, ntrend=1, dates=None, holidays=[], seasPeriods=[], seasHarmComponents=[], ret=['model', 'forecast'], mean_only=False, **kwargs)[source]¶

This is a helpful function to run a standard analysis. The function will: 1. Automatically initialize a DGLM 2. Run sequential updating 3. Forecast at each specified time step

This function has many arguments to specify the model components. Here’s a simple example for a Poisson DGLM:

mod, samples = analysis(Y, X, family="poisson",
                        forecast_start=15,      # First time step to forecast on
                        forecast_end=20,        # Final time step to forecast on
                        k=1,                    # Forecast horizon. If k>1, default is to forecast 1:k steps ahead, marginally
                        prior_length=6,         # How many data points to use in defining prior
                        rho=.5,                 # Random effect extension, increases variance of Poisson DGLM (see Berry and West, 2019)
                        deltrend=0.95,          # Discount factor on the trend component (intercept)
                        delregn=0.95            # Discount factor on the regression component
                        )
Parameters:
  • Y – Array of observations (n * 1)
  • X – Array of covariates (n * p)
  • k – forecast horizon (how many days ahead to forecast)
  • forecast_start – date or index value to start forecasting (beginning with 0).
  • forecast_end – date or index value to end forecasting
  • family – Exponential family for Y. Options: ‘normal’, ‘poisson’, ‘bernoulli’, or ‘binomial’
  • nsamps – Number of forecast samples to draw
  • n – If family is ‘binomial’, this is a (n * 1) array of test size (where Y is the array of successes)
  • model_prior – A prespecified model of class DGLM
  • prior_length – If model_prior is not given, a DGLM will be defined using the first ‘prior_length’ observations in Y, X.
  • ntrend – Number of trend components in the model. 1 = local intercept , 2 = local intercept + local level
  • dates – Array of dates (n * 1). Must be specified if forecast_start and forecast_end are dates.
  • holidays – List of Holidays to be given a special indicator (from pandas.tseries.holiday)
  • seasPeriods – A list of periods for seasonal effects (e.g. [7] for a weekly effect, where Y is daily data)
  • seasHarmComponents – A list of lists of harmonic components for a seasonal period (e.g. [[1,2,3]] if seasPeriods=[7])
  • ret – A list of values to return. Options include: [‘model’, ‘forecast, ‘model_coef’]
  • mean_only – True/False - return the mean only when forecasting, instead of samples?
  • kwargs – Further key word arguments to define the model prior. Common arguments include discount factors deltrend, delregn, delseas, and delhol.
Returns:

Default is a list with [model, forecast_samples]. forecast_samples will have dimension (nsamps by forecast_length by k), where forecast_length is the number of time steps between forecast_start and forecast_end. The output can be customized with the ‘ret’ parameter, which is a list of values to return.

pybats.dglm module¶

class pybats.dglm.dglm(a0=None, R0=None, nregn=0, ntrend=0, nhol=0, seasPeriods=[], seasHarmComponents=[], deltrend=1, delregn=1, delhol=1, delseas=1, interpolate=True, adapt_discount=False, adapt_factor=0.5, discount_forecast=False)[source]¶

Core class of dynamic generalized linear models (DGLMs).

Children include Poisson, Bernoulli, and Binomial DGLMs, as well as the Normal DLM.

A Dynamic Generalized Linear Model (DGLM). Generic Class. Children include Poisson, Bernoulli, and Binomial DGLMs, as well as the Normal DLM.

Parameters:
  • a0 – Prior mean vector
  • R0 – Prior covariance matrix
  • nregn – Number of regression components
  • ntrend – Number of trend components
  • nhol – Number of holiday components
  • seasPeriods – List of periods of seasonal components
  • seasHarmComponents – List of harmonic components included for each period
  • deltrend – Discount factor on trend components
  • delregn – Discount factor on regression components
  • delhol – Discount factor on holiday components (currently deprecated)
  • delseas – Discount factor on seasonal components
  • interpolate – Whether to use interpolation for conjugate parameters (provides a computational speedup)
  • adapt_discount – What method of discount adaption. False = None, ‘positive-regn’ = only discount if regression information is available, ‘info’ = information based, :param adapt_factor: If adapt_discount=’info’, then a higher value adapt_factor leads to a quicker adaptation (with less discounting) on overly uncertain parameters
  • discount_forecast – Whether to use discounting when forecasting

  • X – Regression matrix shape k by p. Each row h has the regression variables for time t+h.
  • nsamps – Number of samples to simulate from the forecast distribution.
Returns:

Samples from the path (joint) forecast distribution from time t+1 through time t+k
update(y=None, X=None, **kwargs)[source]¶

Update the DGLM state vector mean and covariance after observing ‘y’, with covariates ‘X’.

>>> mod.update(y[t], X[t])

Posterior mean and covariance:

>>> [mod.m, mod.C]

You can also access the state vector prior mean and variance for the next time step. The state vector prior mean will be the same as the posterior mean. The variance will be larger, due to discounting.

>>> [mod.a, mod.R]
Parameters:
  • y – Observation
  • X – Regression variables
Returns:

No output; DGLM state vector is updated.
